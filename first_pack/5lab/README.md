# Лабораторная работа 1.5 - Обработка файлов

## Описание
Программа обрабатывает текстовые файлы в зависимости от переданного флага.

## Компиляция
```bash
gcc -std=c99 -Wall -Wextra src/main.c src/file_proc.c src/utils.c -o 5lab
```

## Использование

### Базовые флаги (выходной файл генерируется автоматически)
```bash
./5lab -d input.txt          # Создаст out_input.txt
./5lab -i input.txt          # Создаст out_input.txt
./5lab -s input.txt          # Создаст out_input.txt
./5lab -a input.txt          # Создаст out_input.txt
```

### Флаги с явным указанием выходного файла
```bash
./5lab -d input.txt output.txt
./5lab -i input.txt output.txt
./5lab -s input.txt output.txt
./5lab -a input.txt output.txt
```

### Флаги с 'n' (выходной файл обязателен)
```bash
./5lab -nd input.txt output.txt
./5lab -ni input.txt output.txt
./5lab -ns input.txt output.txt
./5lab -na input.txt output.txt
```

## Флаги

- **-d** / **-nd**: Исключает все арабские цифры (0-9) из файла
- **-i** / **-ni**: Для каждой строки выводит количество латинских букв (A-Z, a-z)
- **-s** / **-ns**: Для каждой строки выводит количество специальных символов (не буквы, не цифры, не пробелы)
- **-a** / **-na**: Заменяет все символы (кроме цифр) на их ASCII код в 16-ричной системе

## Примеры

### Тестовый файл (test_input.txt):
```
Hello World 123!
This is a test 456.
Special chars: @#$%^&*()
```

### Флаг -d (исключить цифры):
```
Hello World !
This is a test .
Special chars: @#$%^&*()
```

### Флаг -i (количество букв в строке):
```
10
11
12
```

### Флаг -s (количество специальных символов):
```
1
1
10
```

### Флаг -a (замена на hex):
```
48656C6C6F20576F726C6420123210A...
```

## Структура кода

- `main.c` - точка входа, обработка аргументов командной строки
- `file_proc.c` / `file_proc.h` - основная логика обработки файлов
- `utils.c` / `utils.h` - вспомогательные функции (проверка символов)

## Обработка ошибок

Программа обрабатывает следующие ошибки:
- Неверное количество аргументов
- Неверный формат флага
- Ошибки открытия файлов
- Ошибки чтения/записи файлов
- Ошибки выделения памяти

Все ошибки возвращаются через статус-коды (enum ProcessStatusCode).
